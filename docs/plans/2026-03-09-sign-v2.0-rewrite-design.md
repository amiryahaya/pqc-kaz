# KAZ-SIGN v2.0 Rewrite Design

## Goal

Rewrite the C KAZ-SIGN library to match Java kaz-pqc-core-v2.0 exactly.
Replace the current 2-component scheme (S1, S2 counter) with the Java v2.0
3-component scheme (S1, S2, S3). This is a clean break — all old keys,
signatures, KAT vectors, and bindings are invalidated.

## Decisions

| Decision | Choice |
|----------|--------|
| Hashing | SHA-256 only, zero-padded to HASH_BYTES (match Java exactly) |
| Scope | Full rewrite — core + all extensions in one pass |
| Compatibility | Clean break, no backward compatibility |
| Prime generation | OpenSSL `BN_generate_prime_ex()` |
| OIDs | New arc `1.3.6.1.4.1.62395.2.*` to distinguish from v1 |

## Algorithm

### System Parameters

Per-level constants from Java `SystemParameters.java`:

- `g1 = 65537`, `g2 = 65539` (all levels)
- `N` — level-specific modulus
- `phiN` — Euler's totient of N
- `Og1N`, `Og2N` — orders of g1, g2 in Z/NZ

### Key Generation

```
s = random in [2^(bitlen(Og1N)-2), Og1N]
t = random in [2^(bitlen(Og2N)-2), Og2N]
v = g1^s * g2^t mod N
PublicKey = v
PrivateKey = (s, t)
```

### Signing

```
h = BigInteger(1, SHA-256(msg) zero-padded to HASH_BYTES)
e1 = random in [2^(bitlen(Og1N)-2), Og1N].nextProbablePrime()
e2 = random in [2^(bitlen(Og2N)-2), Og2N]

S1 = g2^e2 * g1^e1 mod N
S2 = (h - s*S1) * e1^(-1) mod phiN
S3 = h - t*S1 - e2*S2 mod phiN

Signature = (S1, S2, S3)
```

### Verification

```
h = BigInteger(1, SHA-256(msg) zero-padded to HASH_BYTES)
V = v mod N
S1' = S1 mod N

LHS = V^S1' * S1'^S2 * g2^S3 mod N
RHS = (g1*g2)^h mod N

Accept iff LHS == RHS
```

### Message-recovery mode

Signature wire format: `S1 || S2 || S3 || message`

### Public API

All `_ex()` function signatures unchanged. NIST wrapper functions unchanged.
Internal implementation completely replaced.

## Wire Format (KazWire)

Same magic 0x6752, version 0x01. Field layout changes:

| Type | Payload |
|------|---------|
| Public key (TYPE_PUB=0x02) | v (VBYTES) |
| Private key (TYPE_PRIV=0x01) | s (SBYTES) \|\| t (TBYTES) |
| Detached sig (TYPE_SIG_DET=0x10) | S1 \|\| S2 \|\| S3 |
| Attached sig (TYPE_SIG_ATT=0x11) | S1 \|\| S2 \|\| S3 \|\| 0x50 0x01 \|\| len(4) \|\| payload |

### Wire sizes (including 5-byte header)

| Level | PK | SK | Det sig |
|-------|-----|-----|---------|
| 128 | 59 | 37 | 167 |
| 192 | 93 | 55 | 269 |
| 256 | 123 | 69 | 359 |

New arc `2` to distinguish from v1:

- Sign-192: `1.3.6.1.4.1.62395.2.1.2`
- Sign-256: `1.3.6.1.4.1.62395.2.1.3`

## Security Considerations

- N is odd for all levels, so `BN_mod_exp_mont_consttime` works (unlike v1 with even moduli)
- `BN_FLG_CONSTTIME` on secret values s, t, e1, e2
- `kaz_secure_zero()` on all secret intermediates
- `e1.modInverse(phiN)` requires gcd(e1, phiN)=1; guaranteed since e1 is prime and chosen from [2^(lOg1N-2), Og1N] where Og1N divides phiN — the prime e1 won't divide phiN
- No rejection sampling loop (unlike v1), so signing is deterministic in timing
